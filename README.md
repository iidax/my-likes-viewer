# My Likes Viewer

X（旧Twitter）の「いいね」を過去に遡って表示するWebアプリです。

「いいね」のデータは、Xのアーカイブから解析します。
Xのデータの解析はブラウザ上で行うため、安心して利用できます。

### 機能

- 過去の「いいね」の表示[^1]
- 期間指定して「いいね」を検索
- （未実装）アカウントを指定して「いいね」を検索
- （未実装）キーワードを指定して「いいね」を検索
- （未実装）制限されたツイートのメディアの表示

[^1]: 過去1万件を遡って表示できることを確認済み


## 使い方

### STEP1: Xアカウントから「いいね」のデータをダウンロードする

1. X の [データのアーカイブをダウンロード](https://x.com/settings/download_your_data)にアクセスします。
2. 画面の指示に従って、「アーカイブをダウンロード」をリクエストします。
3. 1日程度待機すると、ダウンロードできるようになります。

![Xのアーカイブダウンロードのページ](<docs/images/screen_shot_ 2026-05-13_03.png>)

4. ダウンロードした「アーカイブ」は以下のような構造になっています。その中の`data/like.js`を控えてください。

```
data/
├── twitter-YYYY-MM-DD-***/   # 自身のXのデータのアーカイブ
│   ├── data/
│   │   └── likes.js         # 過去の「いいね」の情報
│   └── archive.html         # 公式のビューワ
```

### STEP2: 「いいね」のデータを解析する

1. 本ウェブサイトの[アップロードページ](https://iidax.github.io/my-likes-viewer/upload)にアクセスします。
2. STEP1で控えた`data/like.js`をアプリにアップロードします。

![アップロードのページ](<docs/images/screen_shot_ 2026-05-13_02.png>)

### STEP3: 「いいね」を表示する

1. 本ウェブサイトの[トップページ](https://iidax.github.io/my-likes-viewer/)にアクセスします。
2. 最新の12件の "いいね" が表示されます。
3. 日付を指定すると、その期間の "いいね" が表示されます。

![アプリのトップページ](<docs/images/screen_shot_ 2026-05-13_01.png>)

### 要件

アーカイブに同梱されているビューワー（HTML）では、likes.jsを動的に読み込んでいないため、「いいね」を確認することができない。
一方で、likes.jsには過去の「いいね」の情報が全て詰まっている。

本アプリは、likes.jsを読み込んで、時期を指定して「いいね」を表示することを目指す。


### インターフェース
- ユーザーは`likes.js`をアップロードする
- 初期状態は「最新」12件を表示する
- ユーザーは`from:YYYY/MM/DD` and `to:YYYY/MM/DD` を指定することができ、12件ずつのページネーションを表示する。（大量取得を避けるため）
- ページを移動するたびに、APIを叩いてデータを取得する。
- 何度も同じページに移動した場合は、APIを叩かずにキャッシュされたデータを表示する。



### 不明点・考慮

- 「いいね」を取得するとき、XのAPIアクセス制限に引っかかるかもしれないので、大量取得は避ける。
